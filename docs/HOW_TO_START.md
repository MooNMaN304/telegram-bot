# Как запустить CI/CD и смотреть логи в Grafana

## Как работает CI/CD?

CI/CD (GitHub Actions) — это робот, который сам делает деплой на сервера после каждого вашего пуша в GitHub.

**Процесс:**

1. Вы делаете `git push` в GitHub
2. GitHub Actions автоматически запускает pipeline
3. Pipeline делает: Тесты → Деплой на Франкфурт → Деплой на Москву

То есть вы НЕ должны сами ходить по SSH и что-то делать на серверах. GitHub Actions сделает всё сам.

## Что сейчас нужно сделать (пошагово)

### Шаг 1: Закоммитить и запушить все изменения в GitHub

Откройте терминал в VS Code и выполните:

```bash
# Добавить все изменённые файлы
git add .

# Создать коммит
git commit -m "feat: add centralized logging with Loki/Promtail/Grafana and CI/CD"

# Отправить в GitHub
git push origin master
```

### Шаг 2: Проверить CI/CD в GitHub

1. Зайдите на https://github.com/MooNMaN304/telegram-bot/actions
2. Увидите запущенный workflow "CI-CD Pipeline"
3. Подождите 5-10 минут, пока он выполнится:
   - ✅ test — пройдут тесты
   - ✅ deploy-frankfurt — автоматически запустит бота + Loki + Grafana + Promtail
   - ✅ deploy-moscow — автоматически запустит воркер + Promtail

### Шаг 3: Открыть Grafana

После успешного деплоя:

1. **Открыть в браузере:** `http://31.76.96.6:3000`
2. **Логин:** `admin`
3. **Пароль:** `admin`

### Шаг 4: Смотреть логи

**Вариант 1 — Explore (простой просмотр):**

1. В Grafana слева меню → иконка лупы (Explore)
2. В выпадающем списке выберите "Loki"
3. В поле ввода введите запрос:
   - `{server_location="moscow"}` — логи с Московского сервера
   - `{service_name="telegram-bot"}` — логи бота
   - `{service_name="celery-worker"}` — логи воркера
   - `{service_name=~"telegram-bot|celery-worker"} |= "ERROR"` — все ошибки
4. Нажмите "Run query" (кнопка с синим щитом)

**Вариант 2 — Дашборд (готовый):**

1. В Grafana слева меню → иконка квадратиков (Dashboards)
2. Выберите "Bot Statistics"
3. Там уже всё настроено:
   - Все логи по сервисам
   - Количество ошибок
   - Логи по локациям
   - Таймлайн действий пользователей
   - Статус задач парсинга

## Если что-то пошло не так

Проверьте статус контейнеров на Франкфурте:

```bash
ssh root@31.76.96.6
docker ps  # посмотреть запущенные контейнеры
docker compose -f /root/telegram-bot/docker-compose.observability.yml logs  # логи стека
```

## Как это работает "под капотом"

```
Вы пишете код → git push → GitHub Actions
                                     ↓
                    ┌────────────────┴────────────────┐
                    ↓                                  ↓
        Сервер Франкфурт                     Сервер Москва
        ┌──────────────────┐               ┌──────────────────┐
        │ Бот → JSON логи   │               │ Celery → JSON логи│
        │ Promtail → Loki   │               │ Promtail → Loki   │
        │ Grafana (веб UI)  │←──HTTP────────│ (отправка логов)  │
        └──────────────────┘               └──────────────────┘
```

После первого пуша всё будет работать автоматически. При каждом следующем пуше GitHub Actions сам обновит код на обоих серверах.

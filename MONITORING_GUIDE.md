# Инструкция по мониторингу и CI/CD

## 1. Централизованное логирование (Grafana Loki)

Все логи с обоих серверов (Франкфурт и Москва) теперь собираются в одном месте.

### Как смотреть логи:

1. Откройте Grafana: `http://<IP_ФРАНКФУРТА>:3000`
2. Логин/Пароль по умолчанию: `admin` / (тот, что вы указали в secrets или `admin`)
3. Перейдите в раздел **Explore** (иконка компаса слева).
4. Выберите источник данных **Loki**.
5. Используйте фильтры (Labels):
   - `{container="telegram_bot_app"}` — логи бота (Франкфурт)
   - `{container="telegram_bot_celery_worker"}` — логи парсинга (Москва)
   - `{server="moscow"}` — все логи из Москвы
   - `{level="ERROR"}` — только ошибки

### Преимущества:

- Логи в формате JSON, что позволяет фильтровать по полям (logger, level, message).
- Не нужно заходить по SSH на серверы, чтобы проверить, почему упал парсинг.

## 2. Дашборды и Аналитика (Grafana)

Grafana позволяет не только смотреть логи, но и строить графики (дашборды).

### Что можно отслеживать:

1. **Ошибки парсинга:** График количества ошибок в минуту. Если график резко пошел вверх — что-то случилось с сайтом кинотеатра.
2. **Активность пользователей:** Сколько команд отправили боту.
3. **Скорость парсинга:** Сколько времени занимает один цикл сбора данных.

### Как создать первый график:

1. В Grafana нажмите **+** (Create) -> **Dashboard**.
2. Нажмите **Add visualization**.
3. Выберите источник **Loki**.
4. В поле запроса (Query) введите: `count_over_time({container="telegram_bot_celery_worker", level="ERROR"}[5m])`.
5. Это покажет количество ошибок за последние 5 минут.

## 3. CI/CD (Автоматический деплой)

Теперь вам не нужно вручную перекидывать файлы.

### Как это работает:

1. Вы делаете изменения в коде на своем компьютере.
2. Делаете `git push origin main`.
3. GitHub Actions автоматически:
   - Запускает тесты.
   - Если тесты прошли, подключается по SSH к обоим серверам.
   - Делает `git pull`.
   - Пересобирает и перезапускает контейнеры.

### Что нужно настроить в GitHub Repository Secrets:

Для работы деплоя добавьте следующие секреты в настройках вашего репозитория на GitHub (Settings -> Secrets and variables -> Actions):

- `FRANKFURT_HOST`: IP сервера в Германии.
- `MOSCOW_HOST`: IP сервера в России.
- `SSH_USERNAME`: Имя пользователя (например, `root` или `ubuntu`).
- `SSH_PRIVATE_KEY`: Ваш приватный SSH ключ для доступа к серверам.

## 4. Структура проекта на серверах

### Франкфурт (Германия):

- Бот, База данных (Postgres), Redis, Grafana, Loki, Promtail.
- Файл запуска: `docker-compose.frankfurt.yml` + `docker-compose.observability.yml`.

### Москва (Россия):

- Celery Worker (парсинг), Browserless, Promtail (отправляет логи во Франкфурт).
- Файл запуска: `docker-compose.moscow.yml`.
